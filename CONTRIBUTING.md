# Contributing to ChitraHarsha

We love your input! We want to make contributing to ChitraHarsha as easy and transparent as possible, whether it's:

- Reporting a bug
- Discussing the current state of the code
- Submitting a fix
- Proposing new features
- Becoming a maintainer

## Development Process

1. **Fork the repo** and create your branch from `main`.
2. **Setup your environment** using `npm install`.
3. **Optimized Build**: Use `npm run prod` to ensure all native assets are synced.
4. **Code Standards**: Maintain the cinematic aesthetic and use optimized collision logic (`checkCollision`).
5. **PR Templates**: Use the provided PR template in `.github/PULL_REQUEST_TEMPLATE.md`.
